# Skill Accumulation Backlog

These are candidate workflows to promote into stable Hermes/Codex skills after they prove useful.

## Ready Soon

- Daily brief: Open-Meteo + RSS + Telegram.
- Game check-in: 21:00 prompt, 22:00 follow-up if blackboard check-in is not marked done.
- Hermes healthcheck: gateway + cron + daily brief + check-in jobs.
- Shared memory update: append distilled durable facts to shared Codex-Hermes Obsidian memory.
- Project status brief: `TASKS.md` + git status + recent blackboard events.

## Product / Money Workflow Candidates

- IT hiring intelligence report: collect job market signals, AI company movement, hiring trends, and company advantages.
- AI company watchlist: track companies, products, hiring pages, funding/news, tech stack hints.
- Student CS assistant: deadlines, coursework summary, study plans, common mistake memory.
- Repo maintenance bot: summarize changes, failing tests, issues, dependency updates, and send Telegram report.

## Promotion Rule

Do not call a workflow a skill until it has:

- one deterministic command or script
- clear input/output contract
- short markdown procedure
- known failure modes
- at least one successful repeat run
