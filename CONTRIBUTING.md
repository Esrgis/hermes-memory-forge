# Contributing

This project packages practical Hermes workstation patterns.

Good contributions:

- bounded tool routes
- Windows-first setup scripts
- Obsidian memory scaffolds
- quota-aware Hermes profiles
- documentation from real failure cases

Avoid:

- committing secrets
- committing personal vault contents
- adding broad autonomous behavior without guardrails
- assuming Linux/macOS paths
- turning this into a large agent framework

Before opening a PR:

1. Run scripts with `-DryRun` where available.
2. Confirm no secrets are in the diff.
3. Keep changes small and reversible.
